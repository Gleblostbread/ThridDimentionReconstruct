# ThridDimentionReconstruct

**Восстановление 3D-структуры по множеству 2D-проекций (Алгоритм Томаси-Канаде)**

Данный проект реализует классический алгоритм факторизации Томази-Канаде (Tomasi-Kanade Factorization) для восстановления трехмерной структуры объекта по его ортогональным проекциям на несколько плоскостей. Проект включает в себя как сам алгоритм реконструкции, так и инструменты для генерации синтетических данных и визуализации результатов.

## 📝 Описание

Задача восстановления 3D-координат по набору 2D-изображений (или проекций) является фундаментальной в компьютерном зрении. Данный проект демонстрирует решение этой задачи для случая ортогональной проекции без шума (или с минимальным шумом), используя метод факторизации измерительной матрицы.

Алгоритм работает в два этапа:
1. **Аффинная реконструкция:** Использование сингулярного разложения (SVD) для получения структуры с точностью до аффинного преобразования.
2. **Метрический апгрейд:** Восстановление евклидовой структуры (с точностью до подобия) путем наложения ограничений ортонормированности на векторы проекционных плоскостей.

## ✨ Возможности

- **Полная реализация алгоритма Томази-Канаде:** От аффинной факторизации до метрического восстановления.
- **Генератор синтетических данных:** Создание случайных облаков точек и их проекций на произвольное количество плоскостей.
- **Интерактивная визуализация:** Построение 3D-графиков с помощью `plotly` для сравнения исходных точек и восстановленной модели.
- **Выравнивание (Alignment):** Использование алгоритма Кабша-Прокруста для совмещения восстановленной модели с эталоном (Ground Truth) или приведения к каноническому виду.
- **Оценка качества:** Расчет проекционной ошибки и визуальное сравнение RMSE.

## ⚡ Быстрый старт

Проект содержит готовый скрипт для демонстрации работы на синтетических данных.

1. Запустите эксперимент:
   ```bash
   python experiment.py
   ```

2. Скрипт загрузит данные из папки `experiment_1`, выполнит реконструкцию и откроет в браузере интерактивный график, где:
   - **Синие точки:** Исходные истинные координаты (Ground Truth).
   - **Красные точки:** Восстановленные координаты.
   - **Зеленые ромбы:** Опорные точки (Anchors), использованные для выравнивания.

## 🛠 Использование

### Генерация данных

Для создания собственных синтетических данных используйте класс `VisAndGen` из модуля `visualition_and_generation.py`.

```python
from visualition_and_generation import VisAndGen
import numpy as np

# Параметры генерации
points_count = 50       # Количество точек в облаке
planes_count = 5        # Количество проекционных плоскостей
cov_matrix = np.eye(3)  # Ковариационная матрица (форма облака)

# Инициализация генератора
generator = VisAndGen(
    points_count=points_count,
    cov_matrix=cov_matrix,
    planes_count=planes_count,
    auto_visualize=True,  # Показать график сразу
    save=True             # Сохранить данные в .npy файлы
)
```

Сохраненные файлы (`points.npy`, `coords_2d.npy` и др.) можно использовать для тестирования алгоритма реконструкции.

### Реконструкция

Основной класс для восстановления структуры — `TomasiKanade` из модуля `tomasi_kanade.py`.

```python
from tomasi_kanade import TomasiKanade
import numpy as np

# Загрузка данных (2D проекции формы [F, N, 2] и опционально Ground Truth)
coords_2d = np.load('path/to/coords_2d.npy')
true_points = np.load('path/to/points.npy')

# Инициализация реконструктора
reconstructor = TomasiKanade(data=coords_2d, true_coords=true_points)

# Полный цикл реконструкции
# anchor_indices - индексы точек, используемых для финального выравнивания
aligned_points = reconstructer.reconstruct(anchor_indices=[0, 1, 2])

# Результат доступен в атрибуте aligned_coordinates
print(f"Форма восстановленных данных: {aligned_points.shape}")
```

#### Пошаговое выполнение

Если вам нужно контролировать процесс поэтапно:

```python
# 1. Аффинная реконструкция
affine_coords = reconstructor.affine_3d_coordinates()

# 2. Метрический апгрейд (переход к евклидовым координатам)
metric_coords = reconstructor.metric_upgrade(er=True) # er=True выведет ошибку реконструкции

# 3. Выравнивание по опорным точкам (Procrustes)
final_coords = reconstructor.align_via_procrustes_anchors(anchor_indices=[0, 1, 2])
```

## Математическая справка

Алгоритм основан на работе *Tomasi, C., & Kanade, T. (1992). Shape and motion from image streams under orthography: a factorization method.*

1. **Измерительная матрица $W$:** Составляется из 2D-координат точек на всех кадрах (плоскостях). После центрирования она имеет ранг 3 для жесткого объекта при ортогональной проекции.
2. **SVD Факторизация:** $W \approx M_{aff} S_{aff}$, где $M_{aff}$ — матрица движения (параметры плоскостей), $S_{aff}$ — структура (3D точки).
3. **Метрическая коррекция:** Находится матрица преобразования $Q$, такая что $M_{metric} = M_{aff}Q$ удовлетворяет условиям ортонормированности строк, соответствующих одной плоскости.
